Release notes

URLStrip 1.0.2 (Build 16)

Desktop beta refresh with Privacy Redirects, Copy Wayback Link support, and the Windows fix for query-less X/Twitter and Reddit privacy redirects, while preserving the earlier Monetization Impact, Confidence, and rule-update hardening work.

What changed

  • Added Monetization Impact and Confidence detail to cleaned-link results on macOS and Windows so people can see the likely commercial value and confidence band of what was removed.
  • Expanded desktop stats on macOS and Windows with Today, 7 Days, 30 Days, and All Time windows plus category, impact, and confidence rollups.
  • Kept the fast cleaning flow lightweight while moving deeper explanation into intentional surfaces and linking to the public tracking methodology page.
  • Fixed the Windows issue where clicking Install Rule Update appeared to do nothing because the install button stayed disabled after a rule check.
  • Preserved user rule-state and custom rules across Windows rule bundle installs, improved error reporting, and added a 30-second timeout so failed installs surface clearly.
  • Added opt-in Privacy Redirect support for X/Twitter and Reddit links, with configurable privacy frontends.
  • Added Copy Wayback Link support so users can copy a local Internet Archive lookup URL without submitting or archiving the page.
  • Fixed the Windows Privacy Redirect path for query-less canonical X/Twitter and Reddit URLs, where the cleaner could previously bail out before rewriting the host.
  • Aligned the shared desktop release metadata on URLStrip 1.0.2 (Build 16) for clean update checks and published hashes.
  • Refreshed the macOS Build 16 DMG with the new Safari extension enabled, restored the app icon in the packaged app, fixed Safari Settings manifest resource warnings, and re-notarized/stapled the artifact.
  • Refreshed macOS Safari extension build to fix privacy-redirect escape links for Redlib/Reddit and X/Twitter frontends.
